# Task Plan: task0003 — develop SKILL.md rewrite (discovery + Step C)

## Goal

The develop orchestrator finds features via integration branches, reads/writes
all state inside the worktree with per-update commits, and completes with a
simplified Step C free of evacuation logic.

## Requirements

FR4, FR5, FR2, NFR1, NFR2

## Scope

### Files to Create
- (none)

### Files to Modify
- `em-workflow/skills/develop/SKILL.md` — Step A discovery, Step B state handling, Step C simplification

## Design

Follows IMPLEMENTATION.md D1/D2/D3 and the Discovery / Doc-commit
conventions.

Document changes:
1. Step A: replace the feature-docs glob with the Discovery semantics
   (branch enumeration `em-workflow/*/integration` + worktree list; branch
   without worktree → re-materialize; multiple → AskUserQuestion, batch
   abort; zero → create-spec). Path argument handling: an existing directory
   argument now means a feature is addressed by name/branch, not by main-tree
   directory.
2. Step B: workflow.yaml is read from and written to the worktree path;
   every status update the orchestrator writes is followed by a
   commit-docs.sh call. The state SSOT statement ("根拠は workflow.yaml の
   status のみ") is unchanged, only its location moves.
3. Verify / retrospect phase sections: artifact paths become
   worktree-relative; retrospect.yaml write ends with a commit.
4. Step C: delete the final-sync step (C.1) and the untracked
   identity-check / `gio trash` / `mv` evacuation block entirely. New flow:
   main-tree cleanliness check (gitignore-guard `.gitignore` line remains the
   only tolerated exception) → merge (or batch PR variant: push +
   `gh pr create`, unchanged) → worktree/branch cleanup → report.
5. Remove the two-layer model wording (main live copy vs integration
   snapshot) wherever it appears.

## Acceptance Criteria (MANDATORY)

- [ ] AC-1: Step A describes branch-based discovery including worktree
  re-materialization, the multiple-branch ask (batch: abort), and the
  zero-branch create-spec route; no feature-docs main-tree glob remains.
- [ ] AC-2: Every orchestrator write of workflow.yaml / feature-docs content
  is followed by an instruction to commit via commit-docs.sh with a
  `docs({feature}):` message.
- [ ] AC-3: Step C contains no evacuation/identity-check logic; its dirty
  handling is: gitignore-guard line tolerated, anything else → abort with
  report; merge and batch-PR behavior otherwise unchanged.
- [ ] AC-4: The document nowhere states that artifacts live untracked in the
  main working tree.
