# Task Plan: task0006 — commit-docs.sh recovery contract + hardening + test alignment

## Goal

commit-docs.sh detects staleness by actual ref movement (not byproduct
proxies), stages artifact deletions safely, resists rename/allowlist bypass,
and its test suite exercises the real contract (all green).

## Design

Rework of review round 1 residual findings (reviews/round1.yaml):
f5084f2afab5d3fe, 16702654a8c83434 (script side), cb2a99347619de01,
e8313f5cad581a23, 33fb54f4ebe662f0, dee0a6aac12247d6, 1839533482c373b9,
0be68a14be06e869. Their descriptions/suggestions are the authoritative
prescriptions; key decisions:

1. **Staleness = ref movement, not byproducts**: capture the branch tip
   BEFORE acquiring the lock (or accept an expected-tip argument); after
   acquiring the lock, compare the current branch tip against the worktree's
   checked-out commit base. Divergence (an external update-ref advance) →
   exit 4. Non-artifact untracked/modified files (verify build/test/format
   byproducts) must NOT trigger exit 4.
2. **Recovery contract stated in the header**: exit 4 means the caller must
   refresh the worktree to the branch tip, re-apply the artifact edits, and
   retry once. (The protocol-side implementation of that loop is task0007's
   scope — this task only specifies the contract precisely in the header.)
3. **Staging hardening**: include tracked deletions under the artifact roots
   (derive deleted paths from git's deleted-file listing); validate staged
   paths with a NUL-delimited, rename-suppressed listing (no parsing of
   porcelain rename arrows) and fail if any staged path falls outside the
   artifact roots.
4. **Tests**: success/contention/git-failure cases write under
   `feature-docs/{feature}/...` (artifact scope); TS-4 drives a REAL
   checkout-free ref advance (update-ref with a tree-changing commit, or an
   actual merge-task.sh invocation) concurrently and asserts both commits
   survive with linear history; add a dedicated die-4 test (stale via
   external update-ref → exit 4; then refresh + retry → success); add the
   TS-5 develop-shaped integration test (branch+worktree creation → doc
   writes/commits → simulated hard reset → main tree clean, committed state
   intact).

## Acceptance Criteria (MANDATORY)

- [ ] AC-1: With non-artifact untracked/modified files present but NO
  external ref advance, commit-docs.sh commits the artifact changes (exit 0)
  — no false die 4.
- [ ] AC-2: After an external checkout-free ref advance with a changed tree,
  commit-docs.sh exits 4 without committing; after refreshing the worktree
  and re-applying doc edits, a retry succeeds and no merged content is
  reverted.
- [ ] AC-3: Deleting a tracked file under an artifact root is staged and
  committed; a staged path outside the artifact roots (including via rename)
  causes a non-zero failure without a commit.
- [ ] AC-4: The full suite `python3 -m unittest discover -s tests` passes,
  including the reworked TS-1/TS-4 tests, the new die-4 test, and the new
  TS-5 integration test.

## Test Notes

- AC-1/AC-2: fixture repos with a second worktree performing the update-ref
  advance (mirror merge-task.sh's mechanism, not a same-worktree commit).
- AC-3: rename staging attempt via git mv from artifact to non-artifact path.
- TS-5 asserts main-repo `git status --porcelain` emptiness (tolerating only
  a gitignore-guard line) and doc-commit survival after the reset.
