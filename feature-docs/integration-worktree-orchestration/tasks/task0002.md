# Task Plan: task0002 — create-spec agent rewrite (early worktree creation)

## Goal

requirements-spec-creator creates the integration branch + worktree at feature
name confirmation and writes every artifact inside the worktree, committing
per update.

## Requirements

FR1, FR2, NFR1

## Scope

### Files to Create
- (none)

### Files to Modify
- `em-workflow/agents/requirements-spec-creator.md` — Phase 3 rewrite + artifact path changes throughout (including the Batch Mode section)

## Design

Follows IMPLEMENTATION.md D1/D2 and the Worktree layout / Doc-commit message
conventions.

Document changes:
1. Phase 3 becomes "create integration branch + worktree": after the feature
   name passes fail-closed validation, create branch
   `em-workflow/{feature}/integration` from base_branch HEAD and its worktree
   at the conventional path, then create `feature-docs/{feature}/` INSIDE the
   worktree. A pre-existing branch of the same name → treat as resume/ask per
   the behavior described in SPEC.md FR1 (batch: abort with report).
2. Phases 4 / 5 / 5.5: REQUIREMENTS.md / SPEC.md / workflow.yaml paths become
   worktree-relative; each phase ends with a commit-docs.sh call
   (`docs({feature}): {summary}` messages), referencing the contract in
   IMPLEMENTATION.md — the agent instruction states WHEN to commit, the
   script defines HOW.
3. Phase 0.5 (test/README.md creation) and any design-system references:
   created inside the worktree, never in the main tree.
4. Batch Mode section: same path/commit rules; no main-tree writes.
5. Remove any instruction implying artifacts live untracked in the main
   working tree.

## Acceptance Criteria (MANDATORY)

- [ ] AC-1: The document instructs creating branch + worktree at Phase 3
  (name confirmed, before any artifact write), including the pre-existing
  branch handling (interactive resume/ask; batch abort).
- [ ] AC-2: Every artifact write instruction (REQUIREMENTS.md, SPEC.md,
  workflow.yaml, test/README.md) resolves to a path inside the integration
  worktree, followed by a commit instruction using commit-docs.sh with a
  `docs({feature}):`-style message.
- [ ] AC-3: The document contains no remaining instruction to write workflow
  artifacts into the main working tree (grep for feature-docs writes outside
  the worktree context finds nothing).
- [ ] AC-4: The Batch Mode section is consistent with the same path/commit
  rules.
