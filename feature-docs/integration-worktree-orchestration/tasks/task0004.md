## Design

Follows IMPLEMENTATION.md D1/D2/D3.

Document changes:
1. implement-phase.md: rewrite the "Branch & Worktree Model" section — the
   integration worktree exists from create-spec (not created in Step I.1);
   the live feature-docs statement ("stays in the MAIN working tree,
   uncommitted") is replaced by the committed-in-worktree model. Step I.1
   loses the docs copy-and-commit and the untracked-docs carry-over; the
   reconcile `reset --hard` refresh stays, justified by the
   everything-committed invariant (NFR2). gitignore-guard pre-step stays.
2. review-phase.md: round records (`reviews/roundN.yaml`), finding
   resolution updates, and any feature-docs reads/writes are worktree paths,
   each write followed by a commit-docs.sh call.
3. batch-mode.md: Step A row of the decision table reflects branch-based
   discovery (multiple branches → abort with report; zero → batch
   create-spec); Step C row loses evacuation wording; the `batch` block in
   workflow.yaml is unchanged (counters only).
4. workflow-schema.md: the "Sibling artifacts" section states artifacts live
   committed in the integration worktree; remove the "while untracked ...
   carried into the integration branch by Step I.1" description.
